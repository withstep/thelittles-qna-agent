# 답변 에이전트(Q&A Agent) 개발 워크플로우

본 문서는 `data/counseling_vectors.db` 에 구축된 벡터 데이터베이스를 활용하여, 리틀약사 상담 내역 기반의 **AI 답변 에이전트**를 개발하기 위한 단계별 워크플로우입니다.

---

## 🏗️ 아키텍처 개요 (RAG 기반)

**RAG (Retrieval-Augmented Generation)** 패턴을 사용하여, 사용자의 질문에 대해 과거의 정확한 상담 내역을 검색한 후, 이를 바탕으로 LLM(대형 언어 모델)이 전문적이고 자연스러운 답변을 생성합니다.

1. **사용자 입력**: 사용자가 건강 고민이나 제품 추천을 질문합니다.
2. **하이브리드 검색(Retrieval)**: SQLite 벡터 DB에서 벡터 유사도 검색과 키워드 검색(FTS5)을 결합하여 가장 유사한 과거 Q&A 컨텍스트를 Top-K개 추출합니다.
3. **프롬프트 구성(Augmentation)**: 추출된 과거 답변 내역과 사용자의 질문을 조합하여 LLM에 전달할 프롬프트를 만듭니다.
4. **답변 생성(Generation)**: LLM이 프롬프트를 분석하여 최종 답변을 생성합니다.

---

## 📅 단계별 개발 워크플로우

### Phase 1: 검색(Retrieval) 모듈 고도화 및 모듈화
기존에 작성된 `build_vector_db.py`의 검색 유틸리티를 실제 서비스에서 사용할 수 있도록 별도의 모듈로 분리하고 고도화합니다.

*   **Task 1-1**: `retriever.py` 모듈 생성
    *   DB 연결 및 임베딩 모델(`paraphrase-multilingual-MiniLM-L12-v2`) 로딩 최적화 (싱글톤 패턴 등 활용).
    *   하이브리드 검색(`hybrid_search`) 함수 캡슐화.
*   **Task 1-2**: 검색 성능 튜닝
    *   Vector Weight와 FTS(Keyword) Weight의 적정 비율 테스트 (예: 0.7 vs 0.3).
    *   반환할 Top-K 개수 선정 (일반적으로 3~5개가 적당).

### Phase 2: LLM 연동 및 프롬프트 엔지니어링
검색된 데이터를 바탕으로 답변을 생성할 LLM을 선택하고 프롬프트를 설계합니다.

*   **Task 2-1**: LLM API 연동
    *   사용할 LLM(OpenAI GPT-4o, Anthropic Claude, Gemini 등) 클라이언트 셋업.
*   **Task 2-2**: 시스템 프롬프트(Persona) 설계
    *   에이전트의 역할(친절하고 전문적인 약사) 부여.
    *   말투 가이드라인 설정 (예: "~해요", "~합니다", 이모지 사용 등).
*   **Task 2-3**: RAG 프롬프트 템플릿 구성
    *   검색된 과거 Q&A 데이터를 참조 정보(`Context`)로 제공.
    *   **제약 조건 설정**: "제공된 과거 상담 내역을 최우선으로 참고하되, 의학적 진단은 피하고 조언 형태로 답변하라."

### Phase 3: 백엔드 API 서버 개발 (FastAPI)
프론트엔드나 다른 플랫폼에서 호출할 수 있는 API 서버를 구축합니다.

*   **Task 3-1**: FastAPI 프로젝트 설정
    *   `main.py` 파일 생성 및 CORS 설정.
*   **Task 3-2**: API 엔드포인트 구현
    *   `POST /api/v1/chat`: 사용자 쿼리를 받아 답변을 스트리밍(Streaming) 또는 JSON 형태로 반환.
    *   `요청 모델`: `{"query": "임산부 오메가3 추천해주세요", "history": [...]}`
    *   `응답 모델`: `{"answer": "...", "sources": [{"id": 1, "title": "..."}]}` (출처 제공 기능).

### Phase 4: 대화형 UI 개발 (옵션/선택)
사용자가 직접 에이전트와 대화할 수 있는 인터페이스를 구성합니다.

*   **Task 4-1**: Streamlit 또는 웹 프론트엔드 구축
    *   빠른 프로토타이핑을 위해 Python `Streamlit` 또는 `Gradio`를 활용한 챗봇 UI 구현.
    *   채팅 창 및 사용자 입력 폼 구현.
*   **Task 4-2**: 출처(Source) 표시 기능
    *   답변 하단에 어떤 과거 상담 내역(`wr_subject`, `ca_name`)을 참고했는지 클릭 가능한 링크나 텍스트로 표시하여 신뢰도 확보.

### Phase 5: 테스트 및 배포 (CI/CD)
*   **Task 5-1**: 엣지 케이스 테스트
    *   데이터베이스에 없는 전혀 엉뚱한 질문에 대한 방어 로직 테스트.
    *   복합적인 증상이나 두 개 이상의 제품을 비교하는 질문 테스트.
*   **Task 5-2**: 운영 환경 배포
    *   Docker 컨테이너화 (`Dockerfile` 작성).
    *   클라우드 플랫폼(AWS, GCP 등) 배포.

---

## 🛠️ 기술 스택 (권장)

*   **Vector DB / Search**: SQLite (FTS5) + `sentence-transformers`
*   **Backend Framework**: `FastAPI` (비동기 처리 및 빠른 API 응답)
*   **LLM API**: `OpenAI` (gpt-4o-mini 등) 또는 `LangChain` / `LlamaIndex` 프레임워크 (선택사항, 직접 구현 권장)
*   **Frontend**: `Streamlit` (빠른 PoC 구축용) 또는 `Next.js`
